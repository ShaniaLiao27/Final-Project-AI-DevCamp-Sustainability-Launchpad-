## 2. 產品願景與用戶故事

**主角**：阿明，48 歲，台北市開三家連鎖手搖飲店，員工 25 人。
**處境**：銀行說明年起貸款要看 ESG 評分；女兒大學念環境工程，跟他說「爸你該做永續了」。但他不知道從哪開始 —— ESG、SDG、Net Zero、碳盤查...術語滿天飛。

**他打開 Sustainability Launchpad**：

1. **首頁兩個按鈕**：「我想學」與「我想寫」
2. **點「我想學」** → 輸入「什麼是 Scope 1, 2, 3？」 → AI 用手搖飲店例子解釋 → 出 3 題小測驗 → 立刻批改
3. **學了 20 分鐘後點「我想寫」** → 輸入公司基本資料 → AI 產出一頁版「永續行動聲明」 → 對應到 4 個 SDG → 雙語版本可直接貼進官網

**Demo Day 講這個故事**，評審秒懂。

## 3. 系統架構

### 3.1 兩大模式 + 七個 Agent

```
              ┌─────────────────────────────────┐
              │  Master: ModeRouter Orchestrator │
              │  （判斷使用者要「學」還是「寫」）│
              └────────────┬────────────────────┘
                           │
            ┌──────────────┴──────────────┐
            ▼                              ▼
    ┌───────────────┐              ┌────────────────┐
    │  學習模式      │              │   生成模式      │
    │  Learn Mode   │              │  Generate Mode │
    └───────┬───────┘              └────────┬───────┘
            │                               │
   ┌────────┼────────┐             ┌────────┼─────────┐
   ▼        ▼        ▼             ▼        ▼         ▼
┌──────┐ ┌──────┐ ┌──────┐    ┌──────┐ ┌──────┐ ┌──────┐
│概念   │ │SME   │ │測驗  │    │重大性 │ │草稿  │ │SDG   │
│解釋   │ │案例  │ │+評分 │    │分析   │ │生成  │ │對應  │
│Agent │ │Agent │ │Agent │    │Agent │ │Agent │ │Agent │
└──────┘ └──────┘ └──────┘    └──────┘ └──────┘ └──────┘
                       \               /
                        ▼             ▼
              ┌──────────────────────────────┐
              │ 共用：FactChecker + Research │
              │ （用 Google Search MCP 驗證） │
              └──────────────────────────────┘
```

### 3.2 各 Agent 角色與 Prompt 要點

| Agent | 角色 | 關鍵指令 |
| --- | --- | --- |
| **ModeRouterAgent** | Master Orchestrator | 從使用者第一句話判斷是「想學概念」還是「想生成內容」，路由到對應 sub-agent |
| **ConceptExplainerAgent** | 學習・解釋者 | 用「中小企業老闆能懂」的語言解釋，避免英文縮寫先講中文，每個概念配一個生活化例子 |
| **SMECaseStudyAgent** | 學習・案例 | 從手搖飲、餐廳、文具批發、製造業等台灣 SME 場景，舉一個對應該概念的真實案例 |
| **QuizGraderAgent** | 學習・測驗 | 出 3 題（1 選擇 + 1 是非 + 1 簡答），等使用者作答後批改並給「下一步建議學什麼」 |
| **MaterialityAdvisorAgent** | 生成・分析 | 根據使用者輸入的產業 + 規模，列出最該揭露的 5 個永續議題（基於 GRI / SASB） |
| **ContentDrafterAgent** | 生成・撰寫 | 寫一頁版「永續行動聲明」或「ESG 自評摘要」，結構：現況 → 承諾 → 行動 → KPI |
| **SDGMapperAgent** | 生成・對應 | 把產出內容對應到 17 個 SDG 中最相關的 3-5 個，附 SDG 圖示編號 |
| **FactCheckerAgent**（共用） | 品質 | 對任何包含「數字」「年份」「法規名稱」的句子，用 Google Search 驗證 |

### 3.3 技術棧（沿用你 repo 不用換）

| 層 | 技術 | 修改點 |
| --- | --- | --- |
| Agents | Google ADK + Gemini 2.5 Flash | 改 prompt + 增加 3-4 個新 agent |
| Backend | FastAPI + Uvicorn | 加一個 `/api/mode` endpoint |
| Frontend | React + Vite | 改首頁變兩個按鈕「我想學 / 我想寫」 |
| 部署 | Cloud Run + Agent Engine | 沿用 repo 的 `deploy.py` 與 `deploy-combined.sh` |
| 工具 | `google_search` + （加分）Notion MCP | 寫完報告可選擇寫進 Notion |

### 4.3 怎麼讓 Agent 用到這些資料源（兩個層級）

**Level 1（簡單，這次 demo 用）**：
寫進 agent 的 system prompt 裡，例如：
```python
RESEARCH_PROMPT = """When researching, prioritize these authoritative sources:
- For SDG: sdgs.un.org
- For Taiwan regulations: moenv.gov.tw, fsc.gov.tw
- For SME guidance: moeasmea.gov.tw
- For ESG news: csr.cw.com.tw
Always cite the source URL. If you cannot find a fact in these sources, say so."""
```
然後配 `google_search` tool，Gemini 就會優先搜這些網站。

**Level 2（進階，有時間再做）**：
做 RAG（檢索式生成）—— 把 GRI Standards 公開版 PDF、台灣金管會指引下載成 PDF，用 Vertex AI Search 建索引，agent 查到的內容更精準。但這次 demo **不需要**。

## 8. 拿到加分的小細節

1. **SDG 圖示用官方版**：UN 有提供免費 SDG icons（[下載](https://www.un.org/sustainabledevelopment/news/communications-material/)），放進前端立刻專業度 +30%
2. **首頁放一個「今日永續小知識」**：每次 reload 隨機顯示一個 ESG 名詞 + 解釋，agent 即時生成，展示 LLM 應用
3. **下載按鈕**：「生成模式」最後讓使用者下載 Markdown / PDF 報告，可以直接帶走，使用者體驗大加分
4. **多國語言**：demo 時切換中 → 英 → 日，評審大概率是英文母語者，這招很有用
5. **使用追蹤儀表板**（如果有時間）：用 ADK 的 callbacks 把每次 agent 呼叫存到 Cloud Logging，最後給一個簡單 dashboard 顯示「總共幫過 X 位中小企業生成 Y 份永續聲明」
